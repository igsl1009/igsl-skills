Glossary — IGSL v2 Terms
Health Metrics
Health Score
Composite score 0–1 for how well a skill is performing. Formula: applied × completion × (1 − 0.5 × fallback). Green ≥0.8, amber ≥0.6, red below threshold.
Applied Rate
Fraction of sessions where this skill was triggered AND successfully applied. Low value means the skill rarely fires.
Completion Rate
Fraction of triggers that led to a successful outcome. Low value means the skill fires but doesn't help complete the task.
Fallback Rate
Fraction of triggers where Claude fell back to default behavior instead of using the skill. High value hurts health score.
Alert Threshold
Minimum acceptable health score. If a skill's health drops below this, an ⚠ alert appears on the overview and health views.
Skill Types (Hardness)
Hard
Always loaded into context every session. Highest token cost but guaranteed availability. Shown in red.
Soft-hook
Loaded automatically when matching keywords appear in the prompt. Conserves tokens — only costs when relevant. Shown in blue.
Explicit-import
Only loaded when directly invoked by name. Maximum token efficiency. Shown in green.
Project-hook
Loaded automatically when working in a matching project directory. Shown in amber.
Memory Node Types
DEC — Decision
A choice that was made (why, when, by whom).
PAT — Pattern
A recurring behavior or approach observed across multiple sessions.
ARCH — Architecture
Structural or design decisions about a system.
ART — Artifact
A file, output, or deliverable that was produced.
LOOP — Open Loop
An unresolved question or pending task. Pulses in the graph. Closed with node.py close.
GEN — General
Miscellaneous notes that don't fit other types.
ERR — Error
An error pattern captured to avoid in future sessions.
Other Concepts
Weight (w)
Relevance score 0–1 for a memory node. Decays over time (each type has a half-life in days). Nodes below w=0.1 are garbage-collected.
Active Index
The subset of memory nodes currently loaded into the session's context window. Shown with a white ring in Memory Net.
EvoGraph
Lineage graph tracking skill version history. Edge modes: FIX (bug fixed), DERIVED (new skill from existing), CAPTURED (pattern formalized into a version).
Token Budget
Total tokens consumed by loading skills + memory. Hard skills always cost tokens; soft skills only when triggered. Lower is better — the goal is progressive disclosure.
Open Loops
Count of LOOP-type memory nodes not yet resolved. Each represents a pending question or task that needs follow-up.
QA Score
Passed/total from the automated test suite (igsl_v2_test_suite.py). Run after any skill change to catch regressions.
